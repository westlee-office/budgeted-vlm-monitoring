\paragraph{Baselines.}
We compare against random scheduling, uniform round-robin scheduling, motion top-$k$, anomaly-score top-$k$, CLIP prompt top-$k$, and dense VLM inspection. The single-signal baselines isolate whether one cheap cue is sufficient. Dense inspection runs the verifier on every stream-time pair and serves as an upper-cost reference rather than a deployable baseline.

\paragraph{Main results.}
Table~\ref{tab:main-results} and Figure~\ref{fig:main-diagnostics} report results at a fixed per-timestep VLM-call budget. \method{} improves event recall by 10.8 percentage points over CLIP prompt top-$k$ and 15.0 points over anomaly top-$k$. It also reduces mean time-to-detect and false alarms. Dense VLM inspection has the highest recall, but costs 7420 GPU-seconds per hour, compared with 1088 for \method.

\paragraph{Detection latency.}
Figure~\ref{fig:main-diagnostics}(c) shows time-to-detect distributions for detected events. A larger fraction of \method{} detections occur within the first 30 seconds after event onset, which is critical in monitoring even when final recall is similar.

\paragraph{Scaling with stream count.}
Figure~\ref{fig:stream-scaling} evaluates the same policy family as the number of simultaneous streams grows. Uniform scheduling degrades rapidly because each stream is revisited less often. CLIP prompt routing is more robust but still misses visually ambiguous events. \method{} preserves a larger fraction of event recall because memory and uncertainty keep borderline streams in the candidate set without repeatedly spending calls on distractors.

\paragraph{Budget interpretation.}
The fixed call-budget setting is stricter than reporting accuracy at a fixed input length: every timestep forces the agent to decide which streams deserve semantic inspection. Under equal calls, all sparse policies spend the same number of VLM queries; improvements therefore come from allocation quality, not from using more expensive perception. Under equal GPU-seconds, dense inspection is penalized because its cost scales linearly with stream count.
